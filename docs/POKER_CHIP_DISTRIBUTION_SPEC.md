### Техническое задание: алгоритм распределения покерных фишек с поддержкой ребаев

#### 1. Назначение

Разработать алгоритм для автоматического распределения покерных фишек между игроками в турнире с возможностью докупки (ребаев). Алгоритм должен учитывать стартовые стеки, лимиты ребаев и доступные номиналы фишек.

#### 2. Входные данные

Программа должна принимать следующие параметры:

* `N` (`int`) — количество игроков;
* `X` (`int`) — максимальное число ребаев на игрока;
* `S` (`int`) — размер стартового стека (в фишках);
* `R` (`int`) — количество фишек, выдаваемых за один ребай;
* `chips_inventory` (`dict`) — словарь с номиналами и количеством фишек в чемодане: `{номинал: количество, ...}`. Пример: `{25: 200, 100: 100, 500: 40, 1000: 20}`;
* `chip_distribution_rules` (`dict`) — правила распределения номиналов в стартовом стеке (доли): `{номинал: доля, ...}`. Пример: `{25: 0.4, 100: 0.3, 500: 0.2, 1000: 0.1}`. Сумма долей должна быть равна 1.0.

#### 3. Выходные данные

Программа должна возвращать:

* `success` (`bool`) — флаг успешности выполнения (True/False);
* `error_message` (`str` или `None`) — сообщение об ошибке, если `success == False`;
* `initial_distribution` (`list`) — список словарей с распределением фишек для каждого игрока на старте: `[{номинал: количество, ...}, ...]`;
* `total_required_chips` (`int`) — общее количество фишек, необходимое для турнира;
* `remaining_chips` (`dict`) — остатки фишек в чемодане после раздачи стартовых стеков: `{номинал: количество, ...}`;
* `rebuy_distribution` (`dict`) — шаблон распределения фишек для одного ребая: `{номинал: количество, ...}`.

#### 4. Алгоритм работы

**Шаг 1. Проверка достаточности фишек**

1. Рассчитать общее количество фишек, которое может быть в игре:
   $$
   Q_{\text{total}} = N \cdot S + N \cdot X \cdot R
   $$
2. Сравнить $Q_{\text{total}}$ с общим количеством фишек в чемодане ($\sum \text{chips\_inventory.values()}$).
3. Если фишек недостаточно, вернуть `success = False` и соответствующее сообщение об ошибке.

**Шаг 2. Расчёт распределения стартовых стеков**

1. Для каждого номинала рассчитать количество фишек на одного игрока в стартовом стеке:
   $$
   n_i = \text{round}(S \cdot \text{chip\_distribution\_rules}[v_i])
   $$
   где $v_i$ — номинал фишки.
2. Проверить, достаточно ли фишек каждого номинала для всех игроков:
   $$
   \text{chips\_inventory}[v_i] \geq N \cdot n_i
   $$
3. Если каких‑то фишек не хватает, скорректировать распределение:
    * уменьшить количество фишек мелких номиналов;
    * увеличить количество фишек крупных номиналов;
    * использовать доступные номиналы для замены.
4. Сформировать `initial_distribution`: для каждого игрока создать словарь `{номинал: количество}` на основе рассчитанного распределения.
5. Обновить `remaining_chips`: вычесть из `chips_inventory` количество фишек, выданных на старте.

**Шаг 3. Формирование шаблона ребая**

1. Использовать тот же принцип распределения, что и для стартового стека, но с учётом остатков фишек.
2. Если каких‑то номиналов не хватает, сформировать эквивалентный набор из доступных номиналов (например, заменить 4 фишки номинала 25 на 1 фишку номинала 100).
3. Сохранить результат в `rebuy_distribution`.

**Шаг 4. Валидация и возврат результата**

1. Проверить, что все игроки получили одинаковый стартовый стек.
2. Убедиться, что остатки фишек корректны.
3. Вернуть `success = True` и все выходные данные.

#### 5. Требования к обработке ошибок

Программа должна обрабатывать следующие ошибки:

* Недостаточное количество фишек в чемодане.
* Некорректные входные данные (отрицательные числа, нулевые значения и т. д.).
* Несоответствие суммы долей в `chip_distribution_rules` единице.
* Отсутствие каких‑либо номиналов в `chips_inventory`.

Для каждой ошибки должно быть выдано понятное сообщение.
