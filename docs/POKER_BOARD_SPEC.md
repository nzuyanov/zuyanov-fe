# Техническое задание: Poker Tournament Board

## 1. Общее описание

Страница `/poker-board` — интерактивная доска для проведения живых покерных турниров. Доска выводится на большой экран (телевизор/проектор), на который смотрят все участники турнира. Управление осуществляется одним оператором.

**Целевое разрешение:** 1920×1080 (Full HD) и выше.  
**Ориентация:** только горизонтальная (landscape).  
**Скролл:** полностью отсутствует. Весь контент помещается на один экран (`overflow: hidden`, `height: 100vh`).  
**Валюта:** рубли (₽).

---

## 2. Визуальный стиль

### 2.1 Общий стиль
- Покерная тематика: тёмные тона, зелёное сукно, карточные мотивы
- Современный, не ретро: чистая типографика, аккуратные тени, без скевоморфизма
- Высокая контрастность для читаемости с расстояния (большой экран)
- Крупные шрифты: основные цифры (блайнды, банк, таймер) должны быть видны с 3–5 метров

### 2.2 Цветовая палитра
- **Фон:** тёмный (чёрный / тёмно-серый, ~`#0D1117` / `#1A1D23`)
- **Акцент основной:** изумрудный / покерный зелёный (~`#10B981` / `#059669`)
- **Акцент вторичный:** золотой (~`#F59E0B` / `#D97706`)
- **Текст:** белый / светло-серый (`#F9FAFB` / `#E5E7EB`)
- **Опасность/внимание:** красный (~`#EF4444`)
- **Карточки игроков:** полупрозрачные тёмные блоки с лёгким свечением

### 2.3 Типографика
- Шрифт без засечек (Inter, или аналог из системы проекта)
- Моноширинный шрифт для числовых значений (таймер, блайнды, суммы) для предотвращения «прыжков» при обновлении цифр

## 3. Экраны и навигация

### 3.1 Навигация
- В углу экрана — кнопка возврата на основной сайт (например, иконка `←` + "Назад")
- При нажатии **во время активной игры**: модальное подтверждение — *"Игра будет поставлена на паузу. Вы уверены, что хотите покинуть страницу?"* с кнопками "Остаться" / "Покинуть" (при выходе игра автоматически ставится на паузу)
- При нажатии **до старта игры**: переход без предупреждения

### 3.2 Стартовый экран
При заходе на `/poker-board`:
- Если есть сохранённая незавершённая игра → модальное окно: *"Обнаружена незавершённая игра (дата, кол-во игроков). Продолжить?"* с кнопками "Продолжить" / "Начать новую"
- Если нет → сразу показать кнопку "Новый турнир" (или аналог), которая открывает модалку настройки

### 3.3 Модальное окно настройки турнира
Полноэкранная модалка (или почти полноэкранная) поверх пустой доски. Содержит все настройки перед стартом. Разделена на логические секции:

#### Секция 1: Основные параметры
| Поле | Тип | Описание | Валидация |
|------|-----|----------|-----------|
| Количество игроков | number input | Число участников турнира | min: 2, max: 20 (разумный лимит) |
| Размер закупа (buy-in) | number input | Сумма в рублях за один закуп | min: 1 |
| Длительность игры | time input | Общее время турнира (ЧЧ:ММ) | min: 10 мин |
| Макс. количество ребаев на игрока | number input | Сколько раз игрок может докупаться в период ребаев | min: 0 |
| Период ребаев | number input (минуты) | Время от начала игры, в течение которого доступны ребаи | min: 0 |

#### Секция 2: Призовые места
Три поля ввода процентов (с дефолтными значениями, редактируемые):

| Место | По умолчанию |
|-------|-------------|
| 1 место | 50% |
| 2 место | 30% |
| 3 место | 20% |

- Валидация: сумма процентов = 100%
- При изменении одного значения — подсвечивать ошибку, если сумма ≠ 100%
- Показывать превью суммы в рублях для каждого места (на основе текущего банка = кол-во игроков × buy-in)

#### Секция 3: Блайнды
| Поле | Тип | Описание |
|------|-----|----------|
| Начальный малый блайнд (SB) | number input | Стартовый размер малого блайнда (в фишках) |
| Начальный большой блайнд (BB) | number input | Стартовый размер большого блайнда (в фишках). По умолчанию = 2 × SB |
| Интервал роста | number input (минуты) | Через сколько минут блайнды растут |
| Множитель роста | number input (десятичный) | На сколько умножаются блайнды (например, 1.5 или 2.0). По умолчанию: 2.0 |

- Блайнды округляются до ближайшего номинала фишки (вверх)
- Показать превью: таблица уровней на основе введённых параметров (первые 5–8 уровней)

#### Секция 4: Фишки
| Поле | Тип | Описание |
|------|-----|----------|
| Номиналы фишек | dynamic list | Список номиналов (например: 25, 50, 100, 500) |
| Количество (общее) | number на каждый номинал | Сколько всего фишек данного номинала имеется |

- Кнопка "Добавить номинал" / "Удалить"
- Автоматический расчёт:
  - Общая стоимость фишек = Σ(номинал × количество)
  - Общая стоимость ÷ кол-во игроков = стоимость фишек на 1 игрока
  - Курс: `1 фишка номиналом X = Y ₽` (рассчитывается как `buy-in ÷ стоимость фишек на 1 игрока × номинал`)
- Показывать рассчитанный курс в реальном времени

#### Секция 5: Игроки
- Таблица / список с полями:
  - Номер (автоинкремент)
  - Имя (текстовое поле, по умолчанию "Игрок 1", "Игрок 2"...)
  - Аватарка (выбор из набора SVG-иконок)
- Количество строк = значению из "Количество игроков"
- **Валидация: имена должны быть уникальными.** При вводе дублирующего имени — подсветка ошибки, кнопка "Старт" заблокирована
- Аватарки: использовать библиотеку SVG-аватарок (например, DiceBear Avatars или аналогичную). Набор из 20–30 уникальных иконок/стилей. Каждому игроку назначается случайная аватарка, с возможностью сменить (клик → выбор из набора)

**Кнопка "Начать турнир":**
- Активна только при валидных данных (все поля заполнены, сумма процентов = 100%, имена уникальны)
- При нажатии — модалка закрывается, запускается игра

## 4. Игровая доска (основной экран)

### 4.1 Компоновка (layout)
Весь экран = `100vw × 100vh`, без скролла. Примерная раскладка:

```
┌──────────────────────────────────────────────────────────┐
│  [← Назад]           POKER TOURNAMENT           [⏸ ][🔊]│  ← Header
├────────────────────────────────┬─────────────────────────┤
│                                │  БЛАЙНДЫ: 50 / 100      │
│                                │  След. уровень: 100/200 │
│     СПИСОК ИГРОКОВ             │  До смены: 04:32        │
│     (карточки)                 │                         │
│                                │  ТАЙМЕР ИГРЫ: 01:24:30  │
│                                │                         │
│                                │  БАНК: 15 000 ₽         │
│                                │                         │
│                                │  ПРИЗОВЫЕ:              │
│                                │  🥇 7 500 ₽             │
│                                │  🥈 4 500 ₽             │
│                                │  🥉 3 000 ₽             │
│                                │                         │
│                                │  КУРС ФИШКИ:            │
│                                │  25 = 5₽ | 100 = 20₽   │
│                                │                         │
│                                │ [Следующая раздача →]   │
├────────────────────────────────┴─────────────────────────┤
│  D: Игрок3    SB: Игрок4    BB: Игрок5                   │  ← Позиции
└──────────────────────────────────────────────────────────┘
```

**Примечание:** это ориентировочная раскладка. Финальное расположение элементов может быть адаптировано для лучшей читаемости и эстетики, но все перечисленные элементы должны присутствовать.

### 4.2 Header (верхняя панель)
- Кнопка "Назад" (слева)
- Заголовок "POKER TOURNAMENT" или кастомное название (по центру)
- Кнопка паузы ⏸ / ▶ (справа)
- Кнопка звука 🔊 / 🔇 (справа, рядом с паузой)

### 4.3 Карточки игроков (левая/центральная зона)
Каждый активный игрок отображается карточкой:

```
┌─────────────────────┐
│ [AVA]  Игрок 1      │
│        Внёс: 3000₽  │
│        Ребаев: 1/3   │
│    [+ Ребай] [Выбыл] │
└─────────────────────┘
```

Содержимое карточки:
- **Аватарка** (SVG иконка)
- **Имя** (кликабельное для редактирования; при редактировании — валидация уникальности)
- **Аватарка** — кликабельна для смены по ходу игры
- **Сумма внесённых средств** (buy-in + ребаи)
- **Счётчик ребаев:** `использовано / максимум`
- **Кнопка "+Ребай":**
  - Видна только если игрок ещё может докупиться
  - В период ребаев: доступна, если использовано < максимума
  - После периода ребаев: доступна ровно 1 раз (add-on), если игрок ещё не воспользовался add-on'ом. Эта единственная допокупка доступна **независимо** от того, сколько ребаев было сделано ранее
  - После использования add-on'а: кнопка скрыта навсегда
- **Кнопка "Выбыл":**
  - Помечает игрока как выбывшего
  - Карточка визуально затемняется / перечёркивается
  - Фиксируется порядок выбывания (для определения итоговых мест)
  - Подтверждение перед выбыванием: *"Игрок X выбывает? Это действие нельзя отменить."*

**Адаптация количества:**
- Карточки должны автоматически масштабироваться / перестраиваться в сетку в зависимости от числа игроков (2–20)
- При большом числе игроков — уменьшение размера карточки, но сохранение читаемости

### 4.4 Информационная панель (правая зона)

#### Блайнды
- **Текущий уровень:** SB / BB крупным шрифтом (акцентный цвет)
- **Следующий уровень:** SB / BB более мелким шрифтом (приглушённый)
- **Таймер уровня:** обратный отсчёт до следующего повышения блайндов (формат ММ:СС)
- При повышении — визуальная анимация (вспышка / пульсация)

#### Таймер игры
- **Общий таймер:** обратный отсчёт оставшегося времени (формат ЧЧ:ММ:СС)
- Крупный, хорошо читаемый
- При оставшемся времени < 5 минут — красный цвет

#### Банк
- **Общая сумма** в банке (в рублях): `кол-во закупов × buy-in`
- Обновляется при каждом ребае

#### Призовые
- Три строки: 1 / 2 / 3 место
- Процент + сумма в рублях
- Пересчитываются при изменении банка (ребаи)

#### Курс фишки
- Отображение курса для каждого номинала: `номинал фишки = X ₽`

#### Кнопка "Следующая раздача"
- Крупная, заметная кнопка
- При нажатии:
  - Дилер, SB, BB сдвигаются на одну позицию по часовой стрелке (по списку активных игроков, пропуская выбывших)
  - Визуальная индикация перехода

### 4.5 Полоса позиций (нижняя часть или визуально на карточках)
- Чёткое отображение текущих ролей:
  - **D** (Dealer) — иконка или бейдж на карточке
  - **SB** (Small Blind) — иконка или бейдж
  - **BB** (Big Blind) — иконка или бейдж
- Роли отображаются прямо на карточках соответствующих игроков (бейджи / метки)
- Дополнительно можно продублировать имена с ролями в нижней панели для наглядности

## 5. Механика игры

### 5.1 Таймеры
- **Таймер игры:** отсчитывает от заданной длительности до 0. При достижении 0 — звуковой сигнал (если звук включён), визуальная индикация "Время вышло"
- **Таймер уровня блайндов:** отсчитывает от интервала роста до 0. При достижении 0 — блайнды повышаются автоматически, звуковой сигнал, таймер перезапускается
- Оба таймера останавливаются на паузе
- Формат: таймер игры — `ЧЧ:ММ:СС`, таймер уровня — `ММ:СС`

### 5.2 Блайнды
- Начальные значения задаются в настройках (SB и BB)
- Растут автоматически по таймеру уровня
- Формула: `новый_SB = текущий_SB × множитель`, `новый_BB = текущий_BB × множитель`
- Округление до ближайшего доступного номинала фишки (вверх)
- При повышении:
  - Звуковой сигнал (если включён)
  - Анимация на доске (подсветка / пульсация блока блайндов)
  - Кратковременное отображение "БЛАЙНДЫ ПОВЫШЕНЫ!" (2–3 секунды)

### 5.3 Ребаи и Add-on
**Период ребаев** (от старта до заданного времени):
- Каждый игрок может сделать до N ребаев (N задано в настройках)
- Каждый ребай = размер buy-in
- При ребае: банк увеличивается, призовые пересчитываются, счётчик ребаев игрока обновляется

**Add-on** (после окончания периода ребаев):
- Каждый игрок получает право на ровно 1 дополнительный закуп
- Доступен **независимо** от количества ребаев, сделанных ранее
- Стоимость add-on = размер buy-in
- После использования — кнопка скрывается
- Визуальное отличие кнопки add-on от обычного ребая (другой цвет / текст "Add-on")

**Индикация на доске:**
- Когда период ребаев активен — на доске отображается: "Ребай-период: ММ:СС осталось"
- Когда период ребаев заканчивается — кратковременное уведомление "РЕБАЙ-ПЕРИОД ЗАВЕРШЁН. ДОСТУПЕН ADD-ON"
- Когда все add-on'ы использованы или период add-on'а пройден — индикация "Дозакупки завершены"

### 5.4 Ротация позиций
- Порядок: Dealer → Small Blind → Big Blind → остальные
- Ротация происходит **вручную** при нажатии оператором кнопки "Следующая раздача"
- При ротации выбывшие игроки пропускаются
- При 2 оставшихся игроках (хедз-ап): Dealer = Small Blind (стандартное правило покера)

### 5.5 Выбывание
- Оператор нажимает "Выбыл" на карточке игрока
- Подтверждение через модалку
- Игрок помечается как выбывший, карточка затемняется
- Фиксируется порядок выбывания (последний выбывший = более высокое место)
- Если осталось 3 игрока — можно визуально подсветить, что это финальный стол
- Если осталось 0 активных игроков (или 1) — игра завершается

### 5.6 Завершение игры
Игра завершается при:
- Истечении времени
- Остался 1 активный игрок
- Ручном завершении оператором (кнопка "Завершить турнир")

При завершении:
- Показать экран результатов:
  - 1, 2, 3 место — имя, аватарка, выигрыш
  - Остальные игроки по порядку выбывания
- Очистить сохранённое состояние из localStorage

## 6. Пауза

- Кнопка паузы в header: переключатель ⏸ / ▶
- При паузе:
  - Все таймеры останавливаются (игра + уровень блайндов)
  - На доске отображается полупрозрачный оверлей "ПАУЗА" крупным текстом
  - Кнопки ребаев и "Выбыл" остаются доступными (оператор может вносить изменения)
  - Кнопка "Следующая раздача" остаётся доступной
- Для возобновления — нажать ▶

## 7. Звук

- Кнопка вкл/выкл звука в header (по умолчанию: выключен)
- Звуковые события:
  - Повышение блайндов
  - Окончание периода ребаев
  - Окончание времени игры (общий таймер = 0)

  - Предупреждение: 1 минута до повышения блайндов

## 8. Автосохранение и восстановление

### 8.1 Автосохранение
- **Хранилище:** `localStorage`
- **Интервал:** каждые 60 секунд
- **Ключ:** `poker-board-state` (или аналогичный)
- **Также сохранять при:**
  - Каждом значимом действии (ребай, выбывание, следующая раздача, пауза)
  - Перед уходом со страницы (`beforeunload`)

### 8.2 Сохраняемое состояние
```
{
  version: 1,
  savedAt: timestamp,
  config: {
    buyIn, maxRebuys, rebuyPeriodMinutes,
    gameDurationMinutes,
    prizes: [50, 30, 20],
    blinds: { startSB, startBB, intervalMinutes, multiplier },
    chips: [{ denomination, totalCount }]
  },
  gameState: {
    status: "playing" | "paused",
    elapsedSeconds,
    currentBlindLevel,
    blindTimerSeconds,
    dealerIndex,
    players: [{
      id, name, avatarId,
      totalContributed, rebuysUsed, addOnUsed,
      isEliminated, eliminationOrder
    }],
    totalPot,
    eliminationCounter
  }
}
```

### 8.3 Восстановление
- При загрузке страницы: проверить `localStorage` на наличие сохранённого состояния
- Если найдено и `status !== "finished"` → предложить восстановить
- При восстановлении:
  - Восстановить все данные
  - Поставить игру на паузу (не запускать таймеры автоматически)
  - Оператор вручную нажимает ▶ для продолжения

## 9. Валидация и ограничения

### 9.1 Настройки
- Все числовые поля: min > 0 (кроме макс. ребаев, который может быть 0)
- Сумма призовых процентов строго = 100%
- Имена игроков: уникальные, не пустые, обрезка пробелов
- Номиналы фишек: уникальные, > 0
- Хотя бы 1 номинал фишки

### 9.2 Игровой процесс
- Нельзя сделать ребай выбывшему игроку
- Нельзя отменить выбывание (подтверждение перед действием)
- При 2 игроках — автоматический переход на хедз-ап правила позиций

## 10. Технические требования

### 10.1 Общее
- Страница `/poker-board` в рамках существующего проекта
- Клиентский рендеринг (вся логика на фронтенде)
- Никаких серверных запросов (кроме статических ресурсов)
- Минимальное разрешение: 1920×1080

### 10.2 Производительность
- Таймеры обновляются каждую секунду (`setInterval` или `requestAnimationFrame`)
- Автосейв не должен вызывать заметных фризов
- Анимации — CSS transitions / animations (не перерисовка всего экрана)

### 10.3 Аватарки
- Использовать библиотеку SVG-аватарок (рекомендация: DiceBear с подходящим стилем, например `adventurer`, `bottts`, `avataaars`, или аналог)
- Набор: минимум 20 уникальных аватарок
- Если библиотека не подходит, допустимо: набор из 20–30 захардкоженных SVG-иконок покерной тематики (карты, масти, фишки, etc.)
- Аватарка выбирается при создании игрока и может быть изменена в любой момент

### 10.4 Зависимости
- Минимум новых зависимостей
- Для аватарок: одна библиотека (DiceBear или аналог)
- Звуки: Web Audio API (без внешних аудиофайлов)

## 11. Краевые случаи

| Ситуация | Поведение |
|----------|-----------|
| Все игроки выбыли кроме одного | Игра завершается, оставшийся = 1 место |
| Время вышло, но игроков > 1 | Показать уведомление, оператор завершает вручную или продолжает |
| Ребай-период = 0 минут | Ребаи в основном периоде недоступны, сразу доступен add-on |
| Макс. ребаев = 0 | Обычные ребаи недоступны, после ребай-периода доступен 1 add-on |
| 2 игрока (хедз-ап) | Dealer = SB, другой игрок = BB |
| Блайнд после умножения не кратен фишке | Округление вверх до ближайшего номинала |
| Оператор перезагрузил страницу | Предложить восстановить игру из localStorage |
| localStorage заполнен | Показать предупреждение, попытаться записать (try/catch) |
| Попытка ввести дублирующее имя | Подсветить ошибку, заблокировать старт / подтверждение |

## 12. Не входит в текущий скоуп

- Мультиязычность (только русский)
- Серверное хранение / синхронизация между устройствами
- Управление с мобильных устройств игроками
- Стриминг / OBS-интеграция
- История прошлых турниров
- Адаптация под мобильные разрешения (только 1920×1080+)
