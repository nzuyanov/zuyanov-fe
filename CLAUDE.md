# CLAUDE.md — Правила проекта

## Проект

Личный сайт-портфолио. Три раздела:

1. **Главная** — визитка-портфолио
2. **Блог** — авторские статьи (markdown-контент)
3. **Приложения** — набор интерактивных мини-приложений для развлечений (покерная доска и т.д.)

## Стек

- Nuxt 4 (SSG — статическая генерация)
- TypeScript (strict)
- Pinia
- Внешний API (отдельный бэкенд)
- Без UI-библиотеки — кастомные компоненты

## Стиль кода

### Общие правила

- Стрелочные функции везде: `const foo = () => {}`, никогда `function foo() {}`
- Табуляция для отступов, не пробелы
- Язык кода (переменные, функции, комментарии в коде) — английский
- Интерфейсный текст (UI, подписи, тексты кнопок) — русский

### TypeScript

- Strict mode, никаких `any`
- Интерфейсы (`interface`) предпочтительнее типов (`type`), если нет union/intersection
- Все пропсы компонентов типизированы через `defineProps<T>()`
- Все эмиты типизированы через `defineEmits<T>()`
- Composables возвращают объект с явной типизацией

### Vue / Nuxt

- `<script setup lang="ts">` всегда
- Порядок в SFC: `<template>` → `<script>` → `<style>`
- Используй auto-imports Nuxt — не импортируй `ref`, `computed`, `useState`, `useRoute` и т.д. вручную
- Composables именуй `use[Name].ts` и размещай в `composables/`
- Компоненты именуй в PascalCase: `AppHeader.vue`, `PokerBoard.vue`

### CSS

- `<style scoped>` по умолчанию
- CSS-переменные для дизайн-токенов (цвета, отступы, радиусы, шрифты)
- Мобильные стили — базовые, десктоп через `@media (min-width: ...)`
- Никаких `!important`
- БЭМ-подход к именованию классов не нужен — используем scoped-стили

- Только тёмная тема, без переключателя

### Правила применения цветов

- `--text-primary` — заголовки, навигация, важный текст
- `--text-secondary` — описания карточек, тело статьи
- `--text-muted` — даты, плейсхолдеры, неактивные элементы
- `--accent` — кнопки, ссылки, иконки активных элементов
- `--accent-soft` + `--accent` — фон + текст для тегов и бейджей
- Белый `#FFFFFF` — только для текста на `--accent` кнопках
- Никогда не используй чистый `#000000`

### Верстка

- Соблюдай семантику, используй section, article и прочие современные HTML теги
- Mobile-first, полностью адаптивная
- Брейкпоинты: `640px` (sm), `768px` (md), `1024px` (lg), `1280px` (xl)
- Современный, стильный, свежий визуал — плавные анимации, продуманные отступы
- Минимализм с характером
- Для заголовков шрифт Montserrat
- Для всего остального текста шрифт Inter
- Используй кавычки-ёлочки «» в тексте
- Все эмодзи должны выглядеть как в iOS
- Не используй атрибут title, везде, где он нужен, добавляй специальный компонент тултипа

### Tone of voice

- Дружественный, живой, как в интерфейсах ozon.ru
- Используй эмодзи в интерфейсных текстах, где уместно
- Обращение к пользователю на «ты»
- Примеры: «Добро пожаловать! 👋», «Тут пока пусто 🤷‍♂️», «Погнали! 🚀»


## API

- Весь обмен с внешним API — через composables: `useApi.ts` как базовый, доменные `use[Entity].ts` поверх
- Используй `useFetch` / `useAsyncData` для SSG-совместимости
- Обработка ошибок API — единообразно, через composable
- Типизируй все ответы API
- В случае ошибки API выводи уведомление с понятным текстом

## Компоненты

- Один компонент — одна ответственность
- Пропсы для входных данных, эмиты для выходных событий
- Переиспользуемые UI-компоненты — в `components/ui/`

## Pinia-сторы

- Используй Setup Stores (composition API стиль)
- Один стор на домен: `usePokerStore`, `useBlogStore`
- Сторы только для разделяемого состояния — локальное состояние в компонентах через `ref`/`reactive`

## Мини-приложения (apps/)

- Каждое приложение — самостоятельная страница в `pages/apps/`
- Компоненты приложения — в `components/apps/[app-name]/`
- У каждого приложения свой стор при необходимости
- Приложения не зависят друг от друга

## Что НЕ делать

- Не ставить зависимости без согласования
- Не менять `nuxt.config.ts` без причины
- Не добавлять inline-стили
- Не использовать `any` в TypeScript
- Не оставлять `console.log` в коммитах
- Не дублировать код — выноси в utils/composables
- Не создавать файлы на русском языке
